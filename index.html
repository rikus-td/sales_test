01. Philosophy
なぜ「コード」で提案するのか？
パワーポイントやPDFは「死んだ情報」です。我々が提供するのは、常に最新で、顧客の行動をトラッキングし、セキュアに管理される「生きたWebアプリケーション」としての提案です。
Single Source

情報の単一化

価格や仕様が変わっても、Gitを更新すれば全員の提案書が瞬時に最新になります。

Analytics

行動の可視化

顧客がどのページを何秒見たか。JSを通じてCRMへ直接データを流し込みます。

Security

完全なアクセス制御

PDFの流出リスクはゼロ。EmailとIPによる厳格な閲覧制限を敷きます。

02. Messaging Architecture
「主張」と「根拠」を1秒で伝える構造
すべてのスライドは、この .message-area から始まります。メインメッセージで結論を断定し、サブメッセージでその理由を補足します。これが全スライド共通の「思考の背骨」となります。

メッセージエリアのコード

このブロックを配置するだけで、読み手は「このページで何が言いたいのか」を瞬時に理解できます。

  <div class="sub-message">それを支える背景や理由</div>
</div>

06. Glassmorphism
透明感がもたらす先進的なUI
.card クラスの最大の特徴は、背後のグリッド線や光が「ぼやけて透ける」すりガラス効果です。
Interactive

Hover Effect

マウスをこのカードに乗せてみてください。
フワッと浮き上がり、青いボーダーと影がハイライトされます。これが「触れる提案書」の質感です。 

/* カードのCSS定義（参考） */
.card {
  background: rgba(255, 255, 255, 0.8);
  backdrop-filter: blur(12px);
  box-shadow: 0 10px 30px rgba(0,0,0,0.05);
  transition: all 0.3s ease;
}
.card:hover {
  transform: translateY(-5px);
  border-color: var(--ta-blue);
}
